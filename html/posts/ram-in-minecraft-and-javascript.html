RAM in Minecraft and JavaScript

2022-08-26

In this post, I will be going through chapter sixteen of "Code: The Hidden Language of Computer Hardware and Software" by Charles Petzold. I will follow the instructions in this chapter to create RAM in Minecraft out of some simple logic gates.

We'll start by creating the assemblage of gates you see in the following picture. This is a level-triggered D-type flip-flop, capable of storing one bit of information.

I'm not going to go into much detail about this circuit, because we've already covered logic gates in a previous post, and the picture above clearly shows how the gates are connected.

When the Clock input is 1, the Q output is the same as the Data input.

When the Clock input is 0, the Q output holds the last value of the Data input.

From now on, we'll refer to the Clock input as Write, because this signal causes the Data In signal to be written to or stored in the circuit. Whenever we want to store the Data In signal in the flip-flop, we make the Write input 1 and then 0 again.

This type of circuit is also called a "latch" because it latches onto data. This is the symbol we'll use to refer to a 1-bit latch:

We'll make an 8-bit latch by lining up eight of them together and connecting the Write signals like this:

This is great because now we can set the Write signal to 1, and then we can store a signal in each of the eight latches. The output of each latch will show us the value that's stored in that latch.

Currently, each latch displays the value it has stored, because each output is connected to its own lamp. We want a way to be able to select which latch we want to see the vimage.pngalue of. To do this, we need to connect the output of each latch to a device that allows us to select which of the latch outputs we want to see. For this, we need an 8-Line-to-1-Line Data Selector. It looks like this:

There's eight Data inputs to this device, and one Output. On the left side of this device, there are three switches which allow us to select which of the eight Data inputs we want to see in the output. Three switches can represent eight different values: 000, 001, 010, 011, 100, 110, and 111.

The 8-to-1 Selector is built from three inverters, eight 4-input AND gates, and an 8-input OR gate, wired together like this:

As you can see, all four inputs must be 1 for the output to be 1.

After that, I connected the output of each latch to the input of each of the AND gates. Then I connected the output of each AND gate to an 8-input OR gate:

I used command blocks to connect the outputs and inputs "wirelessly". This is how I created the arrangement of switches which allow us to select which latch we'd like to see the input of:

To recap, this arrangement of switches allows us to select which of the 8 latches we want to see the value of.

I will demonstrate this now. I will set the Write input of the 8-bit latch to 1. I will set the Data input of latch 7 (labeled D7) to 1. This causes the Data input of the corresponding 4-input AND gate to be 1. After that, I will fly over to the address input (the three switches), and I will close all three switches, which represents the address 111 (we can say that the address of the latch D7 is 111). Now all of the inputs to the D7 latch's corresponding 4-input AND gate are 1, which causes the output to be 1. This causes one of the inputs to the final 8-input OR gate to be 1, which causes the output to be 1. This output represents the value that is held in latch D7.

As you can see, there's quite a lot going on here, but it's really not that difficult to understand. This logic table describes the addresses assigned to each latch. I demonstrated this with latch D7.
