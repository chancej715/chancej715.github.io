Memory

Now that I've shown you how to make a NAND gate, I'll show you how they can be combined to store one bit of memory. The following circuit describes 1-bit of computer memory:

Here it is in Minecraft:

I encourage you to download the schematic here and follow along in Minecraft. Being able to visibily see where the electricity is flowing helps tremendously in understanding how this circuit works. This circuit will be the building block for the following circuits and the RAM itself.

It works like this: the lever on the left corresponds to "i" in the diagram. This is the lever that should be used to input the bit to be remembered. The lever on the right corresponds to "s" in the diagram. In Minecraft I replaced the redstone wire with repeaters and set them to their slowest setting. I also set the time to night. I did this to make it as clear as possible how the bit is travelling through the wire:

I'll follow what the book says, and set the "s" input to 1 and leave the "i" input as 0:

  • As you can see, "i" and "s" go into gate 1. We just turned "s" on, so one input to gate 1 is on, but the other input is off, meaning the output of gate 1 (the wire "a") remains on.
  • The wire "a" and the input "s" are both on and they are inputs to gate 2. Since both inputs to gate 2 are on, the output (the wire "b") is off.
  • Now the wire "b", which is an input to gate 2 is off, therefore the output of gate 2, wire "c" is on.
  • Wire "c" and and "a" are both on, and are inputs to gate 3, so the output of gate 3, "o", is off.

The most important thing to note here is when the "s" input is turned on, the output "o" will reflect the value of the "i" input. I'll demonstrate:

  • Now with "s" still on, I change the "i" to on as well. Now both inputs to gate 1 are turned on, therefore its output (the wire "a") is off.
  • The wire "a" goes to an input of gate 2 and gate 3. Now that "a" is off, the output of gate 2 (wire "b") is on, and the output of gate 3, "o", is on.
  • The output "o" and the wire "b", which are inputs to gate 4, are now both on. This means the output of gate 4 is off, therefore the wire "c" is off.
  • The wire "c" is off now, and it is an input to gate 3. Gate 3 now has two off inputs, so "o" remains on.

Here's what it looks like when I turn "i" off at this point, while "s" remains on:

As you can see, while "s" is on, the output "o" will be the same as the input "i". This isn't very useful by itself, so let's continue. I'll turn "i" back on:

And now I'm going to turn "s" off.

  • When "s" turns off, the wire "a" will be on no matter what the value of "i" is.
  • Notice how the "i" no longer effects the output "o", while "s" is turned off. It's only when "s" is turned on that we can change the value of "o" by changing the value of "i".
  • The wire "a" is an input to gate 2, however the other input to gate 2 is still off, so its output (wire "b") remains on.

This is what happens when I pulse "i" while "s" is off:

Absolutely nothing! This is really exciting, because this means we can now store a bit of information in this circuit, and then come back for it at a later time. We can go for a walk, or watch a movie, and when we come back, our bit will still be represented in this circuit by output "o"!

To get rid of this bit, remember that the "s" input must be on in order for the value of "o" to be the same as "i". So all you need to do is set the value of "s" to on, and then turn off "i".
